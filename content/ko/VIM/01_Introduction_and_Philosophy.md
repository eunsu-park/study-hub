## 1. 간략한 역사

### ed (1969)
Ken Thompson이 Unix용 줄 편집기인 `ed`를 작성했습니다. 파일을 볼 수 없었고 — 명령을 입력하고 결과를 기대하는 방식이었습니다. 오류에 대한 유일한 피드백은 `?`였습니다.

### ex (1976)
Bill Joy가 `ed`를 확장하여 더 강력한 줄 편집 명령을 갖춘 `ex`를 만들었습니다. 이 명령들 중 많은 것이 오늘날 Vim의 `:` 명령줄 모드에서 살아남아 있습니다 (`:s`, `:g`, `:w`, `:q`).

### vi (1976)
Joy는 `ex`에 **비주얼(visual)** 모드를 추가하여 `vi`라는 전체 화면 편집기를 만들었습니다. 처음으로 편집하면서 텍스트를 볼 수 있게 된 것입니다. 이름 자체가 "visual(시각적)"을 의미합니다. 이 때문에 Vim의 명령줄 모드를 "Ex 모드"라고도 부릅니다.

### Vim (1991)
Bram Moolenaar가 주요 개선 사항을 추가한 vi의 클론인 **Vi IMproved**를 출시했습니다:
- 다중 단계 실행취소 (vi는 단일 실행취소만 가능)
- 선택을 위한 비주얼 모드(Visual mode)
- 플러그인 시스템과 스크립팅 (Vimscript)
- 분할 창과 탭
- 구문 강조
- 광범위한 도움말 시스템 (`:help`)

### Neovim (2014)
현대화에 초점을 맞춘 Vim의 커뮤니티 포크:
- 일급 스크립팅 언어로서의 Lua (Vimscript와 함께)
- 내장 LSP 클라이언트 (Language Server Protocol)
- 내장 터미널 에뮬레이터
- 구문 파싱을 위한 Treesitter
- 비동기 작업 제어(Asynchronous job control)
- 기본 설정의 개선

## 2. 모달 편집 철학

> **비유 — 기어 변속**: Vim의 모드를 자동차의 기어 변속기처럼 생각해 보세요. 노멀 모드(Normal mode)는 *주행(drive)* — 텍스트를 이동 중입니다. 입력 모드(Insert mode)는 *주차(park)* — 한 곳에 정지해 텍스트를 추가합니다. 비주얼 모드(Visual mode)는 *후진(reverse)* — 앞뒤로 선택합니다. 주차 상태에서 주행하려 하지 않듯이, 입력 모드에서 탐색하려 하지 않습니다. 모드 전환은 기어를 바꾸는 것만큼 자연스러운 일입니다.

대부분의 편집기는 단일 모드를 가집니다: 타이핑하면 글자가 나타납니다. Vim은 다릅니다 — **다중 모드**를 가지며, 각각은 서로 다른 작업에 최적화되어 있습니다.

### 왜 모드인가?

근본적인 통찰: **코드를 읽고, 탐색하고, 수정하는 데 처음부터 새 텍스트를 입력하는 것보다 훨씬 더 많은 시간을 보냅니다.**

편집 행동 연구에 따르면 프로그래머는 대략:
- **70%** 의 시간을 읽기와 탐색에 사용
- **20%** 기존 텍스트 수정
- **10%** 새 텍스트 입력

단일 모드 편집기는 그 10%에 최적화되어 있습니다. Vim은 90%에 최적화합니다.

### 핵심 모드

| 모드 | 목적 | 진입 방법 | 시각적 표시 |
|------|------|-----------|------------|
| **노멀(Normal)** | 텍스트 탐색 및 조작 | `Esc` (또는 `Ctrl-[`) | 표시 없음 / `-- NORMAL --` |
| **입력(Insert)** | 새 텍스트 입력 | `i`, `a`, `o` 등 | `-- INSERT --` |
| **비주얼(Visual)** | 텍스트 영역 선택 | `v`, `V`, `Ctrl-v` | `-- VISUAL --` |
| **명령줄(Command-line)** | 명령 실행 | `:`, `/`, `?` | 하단의 `:` 프롬프트 |

노멀 모드는 **홈 베이스**입니다. 항상 작업 사이에 노멀 모드로 돌아옵니다. 이것이 가장 중요하게 길러야 할 습관입니다.

### 모드의 효율성

전통적인 편집기에서 단어를 삭제하려면:
1. 마우스로 손을 옮김 → 단어를 더블클릭 → Delete 키 누름

Vim(노멀 모드)에서:
1. `dw` (단어 삭제) 입력 — 두 번의 키 입력, 손이 홈 행을 벗어나지 않음

전통적인 편집기에서 5줄을 삭제하려면:
1. 첫 번째 줄 시작 클릭 → 다섯 번째 줄 끝 Shift+클릭 → Delete

Vim에서:
1. `5dd` 입력 — 세 번의 키 입력

## 3. 2026년에 Vim을 배워야 하는 이유

### 범용적 가용성
Vim(또는 최소한 vi)은 사실상 모든 Unix/Linux 시스템에 설치되어 있습니다. SSH로 서버에 접속하면 Vim이 있습니다.

### 속도
숙달되면 Vim 사용자는 GUI 편집기 사용자보다 훨씬 빠르게 텍스트를 편집합니다. 키보드 전용 워크플로우는 지속적인 손-마우스 컨텍스트 전환을 제거합니다.

### 조합 가능성(Composability)
Vim의 명령은 하나의 **언어**를 형성합니다: 연산자(동사)가 모션(명사)과 결합합니다. 문법을 익히면 복잡한 편집을 간결하게 표현할 수 있습니다. [레슨 5](./05_Operators_and_Composability.md)에서 깊이 있게 배울 것입니다.

### 인체공학(Ergonomics)
손이 홈 행에 머뭅니다. 화살표 키에 손을 뻗거나 마우스를 사용할 필요가 없습니다. 수 시간의 편집에 걸쳐 피로가 줄어듭니다.

### 어디서나 사용 가능한 Vim 키 바인딩
Vim을 직접 사용하지 않더라도 Vim 키 바인딩은 다음에서 사용 가능합니다:
- VS Code (Vim 익스텐션)
- JetBrains IDEs (IdeaVim)
- Sublime Text (Vintage 모드)
- 웹 브라우저 (Vimium)
- 터미널 쉘 (bash/zsh vi 모드)
- Jupyter 노트북
- 그 외 다수

Vim을 배우는 것은 여러 도구에 걸쳐 전이되는 투자입니다.

## 4. vi vs Vim vs Neovim

| 기능 | vi | Vim | Neovim |
|------|-----|-----|--------|
| 출시 연도 | 1976 | 1991 | 2014 |
| 다중 단계 실행취소 | 없음 | 있음 | 있음 |
| 구문 강조 | 없음 | 있음 | 있음 (Treesitter) |
| 플러그인 시스템 | 없음 | Vimscript | Vimscript + Lua |
| LSP 지원 | 없음 | 플러그인을 통해 | 내장 |
| 비동기 작업 | 없음 | Vim 8+ | 있음 |
| 내장 터미널 | 없음 | Vim 8+ | 있음 |
| GUI 변형 | 없음 | gVim | 다수 (Neovide 등) |
| 설정 파일 | `.exrc` | `.vimrc` | `init.lua` / `init.vim` |

**어떤 것을 선택해야 할까요?**

- **기초 학습**: Vim 또는 Neovim 모두 가능 — 핵심 편집 개념은 동일
- **서버 관리**: Vim (더 널리 사전 설치됨)
- **모던 개발 환경**: Neovim (LSP, Treesitter, Lua 생태계)
- **이 가이드**: 둘 다 다룹니다. 레슨 1–13은 둘 다에 적용됩니다; 레슨 14는 Neovim 전용 기능에 초점을 맞춥니다

## 5. 설치

### macOS

```bash
# Vim (기본 설치됨, 하지만 구버전인 경우가 많음)
vim --version | head -1

# Homebrew로 최신 Vim 설치
brew install vim

# Neovim 설치
brew install neovim
```

### Ubuntu/Debian

```bash
sudo apt update
sudo apt install vim        # Vim
sudo apt install neovim     # Neovim
```

### Fedora/RHEL

```bash
sudo dnf install vim-enhanced   # Vim
sudo dnf install neovim         # Neovim
```

### Windows

```bash
# Chocolatey를 통해
choco install vim
choco install neovim

# Scoop을 통해
scoop install vim
scoop install neovim

# 또는 https://www.vim.org/download.php 에서 다운로드
```

### 설치 확인

```bash
vim --version | head -1
# VIM - Vi IMproved 9.x

nvim --version | head -1
# NVIM v0.10.x
```

## 6. 첫 번째 세션: vimtutor

Vim을 배우는 가장 좋은 방법은 내장 튜토리얼입니다:

```bash
vimtutor
```

이 명령은 기본 명령을 대화형으로 안내하는 특수 텍스트 파일을 엽니다. 약 25-30분이 소요되며 다음을 다룹니다:

- 커서 이동 (hjkl)
- 입력 모드 진입 및 종료
- 텍스트 삭제
- 실행취소 및 다시실행
- 기본 파일 조작 (저장, 종료)

**권장 사항**: 첫 주 동안 최소 3번 `vimtutor`를 실행하세요. 매번 동작을 더 깊이 내면화하게 됩니다.

### 처음에 기대할 것

Vim을 처음 열면 (vimtutor가 아닌) 다음과 같은 화면을 볼 것입니다:

```
~
~
~                    VIM - Vi IMproved
~
~                     version 9.1
~                 by Bram Moolenaar
~
~            Vim is open source and freely distributable
~
~                    type  :q<Enter>       to exit
~                    type  :help<Enter>    for help
~
~
```

`~` 문자는 파일 끝을 넘어선 줄(빈 버퍼)을 나타냅니다. 당황하지 마세요 — 이제 `:q` 다음에 Enter를 누르면 종료된다는 것을 알았습니다!

## 7. 학습 곡선

Vim의 학습 곡선은 종종 과장되지만, 실재합니다. 기대할 것은 다음과 같습니다:

```
생산성
    ^
    |        전통적인 편집기
    |       ╱‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
    |      ╱
    |     ╱
    |    ╱         Vim
    |   ╱      ╱‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
    |  ╱      ╱
    | ╱   ╱‾‾╱
    |╱  ╱
    |  ╱    ← 침체기 (1-2주)
    | ╱
    +──────────────────────────────> 시간
     1주   2주   4주   2개월 이후
```

### 학습 곡선 극복 팁

1. **한 번에 다 바꾸지 마세요** — 작은 작업부터 Vim을 사용하세요 (커밋 메시지, 설정 파일)
2. **점진적으로 학습하세요** — 더 추가하기 전에 몇 가지 명령을 숙달하세요
3. **치트시트를 사용하세요** — 모니터 옆에 하나를 두세요 (`examples/VIM/10_vim_cheatsheet.md` 참조)
4. **마우스를 멀리하세요** — 강제로 키보드 명령을 사용하세요
5. **암기하지 말고 이해하세요** — Vim의 명령은 목록이 아니라 언어입니다

## 연습 문제

### 연습 문제 1: Vim 종료하기

터미널에서 `vim`을 실행하여 Vim을 엽니다. 이제 빈 버퍼를 바라보며 노멀 모드에 있습니다. 저장하지 않고 Vim을 종료하세요.

<details>
<summary>정답 보기</summary>

`:q`를 입력하고 `Enter`를 누릅니다. 버퍼가 비어 있고(변경 사항 없음) 아무것도 수정하지 않았으므로 Vim이 즉시 종료됩니다.

실수로 키를 눌러 변경이 발생했다면 `:q!`를 사용하여 저장 없이 강제 종료합니다.

</details>

### 연습 문제 2: Vim 모드 식별하기

레슨 내용을 바탕으로 각 행동에 해당하는 올바른 Vim 모드를 연결하세요:

1. 함수 본체에 새 코드를 입력하고 있습니다.
2. 버그를 찾기 위해 줄 사이를 이동하고 있습니다.
3. `:w`를 실행하여 파일을 저장하고 있습니다.
4. 세 줄을 복사하기 위해 강조 표시하고 있습니다.

<details>
<summary>정답 보기</summary>

1. **입력 모드(Insert mode)** — 새 텍스트를 입력하고 있습니다.
2. **노멀 모드(Normal mode)** — 탐색 중이며, 노멀 모드는 이동을 위한 홈 베이스입니다.
3. **명령줄 모드(Command-line mode)** — `:w`는 `:` 프롬프트에서 입력하는 Ex 명령입니다.
4. **비주얼 모드(Visual mode)** — 텍스트 선택은 비주얼 모드에서 이루어집니다(`V`는 줄 단위 선택).

</details>

### 연습 문제 3: 편집 시간 분포

레슨에서는 프로그래머가 세 가지 활동에 대략 70% / 20% / 10%의 편집 시간을 사용한다고 설명합니다. 보지 않고 그 세 가지 활동을 떠올리고, 이 분포가 Vim의 모달 설계를 어떻게 정당화하는지 설명해 보세요.

<details>
<summary>정답 보기</summary>

- **70%** 코드 읽기와 탐색
- **20%** 기존 텍스트 수정
- **10%** 처음부터 새 텍스트 입력

Vim 설계의 정당성: 전통적인 단일 모드 편집기는 10%(입력)에만 최적화되어 있습니다. 노멀 모드에서 탐색과 조작을 분리함으로써, Vim은 90% 전체(읽기 + 수정)에 최적화합니다. 모달 설계는 가장 일반적인 편집 활동에 보상을 주는 의도적인 트레이드오프입니다.

</details>

### 연습 문제 4: vi vs Vim vs Neovim 기능 비교

레슨의 표를 참고하지 않고 다음에 답해 보세요:

1. 다중 단계 실행취소를 처음 도입한 편집기는?
2. LSP 지원이 기본 내장된(플러그인 불필요) 편집기는?
3. 각 편집기가 사용하는 설정 파일은?

<details>
<summary>정답 보기</summary>

1. **Vim** (1991) — `vi`는 단일 단계 실행취소만 가능했습니다.
2. **Neovim** — Vim은 LSP를 위해 플러그인(vim-lsp, coc.nvim 등)이 필요하지만, Neovim은 내장 LSP 클라이언트를 갖추고 있습니다.
3. 설정 파일:
   - `vi`: `.exrc`
   - `Vim`: `.vimrc`
   - `Neovim`: `init.lua` 또는 `init.vim`

</details>

### 연습 문제 5: vimtutor 실행 후 되돌아보기

터미널에서 `vimtutor`를 실행하고 처음부터 끝까지 완료하세요. 완료 후 다음에 답하세요:

1. 커서를 한 줄 아래로 이동하는 키 입력은?
2. 커서 아래 문자를 삭제하는 명령은?
3. 한 번에 저장하고 종료하는 명령은?

<details>
<summary>정답 보기</summary>

1. `j`는 커서를 아래로 이동합니다(기억법: "j"의 꼬리가 아래로 내려가는 것처럼).
2. `x`는 커서 아래 문자를 삭제합니다(타자기에서 글자를 "지우는" 것처럼).
3. `:wq`는 파일을 쓰고(저장) 종료합니다. 또는 노멀 모드에서 `ZZ`도 동일하게 작동합니다.

</details>
