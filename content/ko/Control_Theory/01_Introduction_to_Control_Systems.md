# 레슨 1: 제어 시스템 입문

## 학습 목표

- 제어 시스템(control system)이 무엇이며 왜 필요한지 정의한다
- 개루프 제어(open-loop control)와 폐루프(피드백) 제어(closed-loop/feedback control)를 구별한다
- 피드백 제어 시스템의 구성 요소를 파악한다
- 성능과 견고성(robustness) 향상에서 피드백의 역할을 이해한다
- 공학 및 과학 전반에 걸친 제어 이론의 응용 분야를 살펴본다

## 1. 제어 시스템이란?

**제어 시스템(control system)**은 원하는 출력을 얻기 위해 동적 시스템(dynamic system)의 동작을 관리, 명령 또는 조절하는 구성 요소들의 상호 연결체입니다.

**일상 속 예시:**
- 실내 온도를 조절하는 온도 조절기(thermostat)
- 차량 속도를 유지하는 크루즈 컨트롤(cruise control)
- 물체에 손을 뻗는 인간의 손 동작 (생물학적 피드백)
- 항공기의 비행 경로를 유지하는 자동 조종 장치(autopilot)

제어 이론의 핵심 질문: *원하는 방식으로 동작해야 하는 시스템(플랜트, plant)이 주어졌을 때, 그 동작을 달성하기 위한 제어기(controller)를 어떻게 설계하는가?*

## 2. 개루프 제어와 폐루프 제어

### 2.1 개루프 제어(Open-Loop Control)

**개루프(open-loop)** 시스템은 실제 출력을 측정하지 않고 미리 계산된 입력을 적용합니다.

```
Reference → [Controller] → [Plant] → Output
```

**예시:** 타이머가 달린 토스터. 시간을 설정하지만, 토스터는 빵이 실제로 구워졌는지 확인하지 않습니다.

**특징:**
- 간단하고 저렴하다
- 외란(disturbance)이나 모델 오차를 보상할 수 없다
- 플랜트에 대한 정확한 사전 지식이 필요하다
- 피드백에 의한 안정도 문제가 없다

### 2.2 폐루프(피드백) 제어(Closed-Loop/Feedback Control)

**폐루프(closed-loop)** 시스템은 실제 출력을 측정하고 **오차(error)**(목표값과 실제값의 차이)를 이용해 입력을 조절합니다.

```
Reference →(+)→ [Controller] → [Plant] → Output
             ↑                           |
             └──── [Sensor] ←───────────┘
                    (feedback)
```

**예시:** 온도 조절기. 현재 온도를 측정하고, 설정값과 비교한 뒤, 그에 따라 히터를 켜거나 끕니다.

**특징:**
- 외란과 모델 불확실성을 보상한다
- 파라미터 변동에 대한 민감도를 줄인다
- 설계가 잘못되면 불안정해질 수 있다
- 개루프보다 복잡하다

### 2.3 비교

| 특성 | 개루프(Open-Loop) | 폐루프(Closed-Loop) |
|------|-------------------|---------------------|
| 외란 거부(Disturbance rejection) | 나쁨 | 좋음 |
| 파라미터 민감도 | 높음 | 낮음 |
| 복잡도 | 낮음 | 높음 |
| 안정도 위험 | 없음 | 가능성 있음 |
| 정확도 | 모델에 의존 | 자기 수정 |
| 비용 | 낮음 | 높음 |

## 3. 피드백 제어 시스템의 구성 요소

### 3.1 표준 블록 선도(Block Diagram)

```
         e(t)        u(t)         y(t)
r(t) →(+)→ [Controller] → [Plant] → Output
        ↑    (G_c)          (G_p)      |
        |                              |
        └────── [Sensor (H)] ←────────┘
                  b(t)
```

| 기호 | 이름 | 설명 |
|------|------|------|
| $r(t)$ | 기준 신호(Reference, 설정값) | 원하는 출력 |
| $e(t)$ | 오차 신호(Error signal) | $e(t) = r(t) - b(t)$ |
| $u(t)$ | 제어 신호(Control signal, 액추에이터 입력) | 제어기의 출력 |
| $y(t)$ | 플랜트 출력(Plant output) | 시스템의 실제 출력 |
| $b(t)$ | 피드백 신호(Feedback signal) | 측정된 출력 (스케일 조정 가능) |
| $G_c$ | 제어기(Controller) | 오차를 처리하여 제어 동작을 생성 |
| $G_p$ | 플랜트(Plant, 공정) | 제어 대상 시스템 |
| $H$ | 센서(Sensor, 피드백 요소) | 출력을 측정 |

### 3.2 외란과 잡음

실제 시스템은 다음의 영향을 받습니다:
- **외란(Disturbances)** $d(t)$: 플랜트를 교란하는 외부 입력 (바람, 부하 변동 등)
- **측정 잡음(Measurement noise)** $n(t)$: 피드백 신호를 오염시키는 센서 불완전성

```
                    d(t)
                     ↓
r(t) →(+)→ [G_c] →(+)→ [G_p] →(+)→ y(t)
        ↑                         ↑ n(t)
        └──── [H] ←──────────────┘
```

잘 설계된 제어기는 잡음을 과도하게 증폭하지 않으면서 외란을 거부합니다.

## 4. 제어 시스템 성능 목표

제어기는 흔히 서로 경쟁하는 여러 목표를 동시에 만족하도록 설계됩니다:

### 4.1 안정도(Stability)

시스템은 반드시 **안정(stable)**해야 합니다. 즉, 유계 입력(bounded input)에 대해 유계 출력(bounded output)이 나와야 하고, 섭동(perturbation) 후 평형 상태로 복귀해야 합니다. 이는 *가장 근본적인 요구 조건*입니다. 불안정한 시스템은 쓸모없을 뿐만 아니라 잠재적으로 위험합니다.

### 4.2 추종(Tracking)

출력 $y(t)$는 기준 신호 $r(t)$를 가능한 한 정밀하게 따라야 합니다:
- 일반적인 기준 신호(계단, 램프)에 대해 **영 정상 상태 오차(zero steady-state error)**
- **빠른 과도 응답(fast transient response)** (짧은 상승 시간, 정착 시간)
- **최소 초과(minimal overshoot)**

### 4.3 외란 거부(Disturbance Rejection)

제어기는 외부 외란이 출력에 미치는 영향을 최소화해야 합니다.

### 4.4 견고성(Robustness)

시스템은 다음 상황에서도 양호한 성능을 유지해야 합니다:
- 플랜트 파라미터의 불확실성
- 모델링되지 않은 동특성(unmodeled dynamics)
- 운전 조건의 변동

### 4.5 민감도(Sensitivity)

**민감도(sensitivity)**는 파라미터가 변할 때 시스템 동작이 얼마나 변하는지를 나타냅니다. 피드백은 민감도를 줄입니다. 이것이 피드백의 주요 장점 중 하나입니다.

**감도 함수(sensitivity function)**:

$$S(s) = \frac{1}{1 + G_c(s) G_p(s) H(s)}$$

**루프 이득(loop gain)** $L(s) = G_c(s) G_p(s) H(s)$이 크면 $S(s) \approx 0$이 되어 출력이 플랜트 변동에 둔감해집니다.

| 연도 | 이정표 |

## 6. 제어 시스템의 분류

### 6.1 신호 유형에 따른 분류

- **연속 시간(Continuous-time)**: 연속 시간 $t$에서 신호와 시스템이 동작
- **이산 시간(Discrete-time)**: 시간 간격 $T$마다 신호가 샘플링됨 (디지털 제어)

### 6.2 선형성에 따른 분류

- **선형(Linear)**: 중첩(superposition) 원리 성립 — 고전 및 현대 제어 이론의 주요 대상
- **비선형(Nonlinear)**: 중첩 원리 불성립 — 특수 기법 필요 (리아푸노프(Lyapunov), 슬라이딩 모드(sliding mode), 피드백 선형화(feedback linearization))

### 6.3 입출력 수에 따른 분류

- **SISO**: 단일 입력, 단일 출력(Single-Input, Single-Output) (고전 제어)
- **MIMO**: 다중 입력, 다중 출력(Multiple-Input, Multiple-Output) (현대 상태 공간 기법)

### 6.4 시변성에 따른 분류

- **시불변(Time-invariant, LTI)**: 파라미터가 시간에 따라 변하지 않음
- **시변(Time-varying)**: 파라미터가 변함 — 예: 연료를 소비하며 질량이 줄어드는 로켓

## 7. 수학적 예비 지식 복습

제어 이론은 미분 방정식과 라플라스 변환 개념을 기반으로 합니다. 가장 핵심적인 내용을 간략히 복습합니다:

### 7.1 선형 상미분 방정식(Linear ODE)

선형 시불변(LTI) 시스템은 다음으로 표현됩니다:

$$a_n \frac{d^n y}{dt^n} + a_{n-1} \frac{d^{n-1} y}{dt^{n-1}} + \cdots + a_1 \frac{dy}{dt} + a_0 y = b_m \frac{d^m u}{dt^m} + \cdots + b_0 u$$

### 7.2 라플라스 변환(Laplace Transform)

라플라스 변환은 미분 방정식을 대수 방정식으로 변환합니다:

$$\mathcal{L}\{f(t)\} = F(s) = \int_0^\infty f(t) e^{-st} \, dt$$

핵심 성질: $\mathcal{L}\left\{\frac{d^n f}{dt^n}\right\} = s^n F(s) - s^{n-1}f(0) - \cdots - f^{(n-1)}(0)$

초기 조건이 영(zero)이면 미분이 $s$의 곱셈으로 변환되어, LTI 시스템 해석이 순수한 대수 연산이 됩니다.

### 7.3 전달 함수(Transfer Function) 미리 보기

**전달 함수(transfer function)**는 초기 조건이 영일 때 라플라스 변환된 출력과 입력의 비입니다:

$$G(s) = \frac{Y(s)}{U(s)}$$

이 개념은 레슨 3-10의 모든 내용에서 핵심이며, 레슨 3에서 완전히 전개됩니다.

## 연습 문제

### 연습 1: 시스템 요소 파악

각 시스템에서 기준 입력, 플랜트, 센서, 외란, 제어기를 파악하라:

1. 실내 온도를 22°C로 유지하는 에어컨 시스템
2. 고속도로에서 차선을 유지하는 자율 주행 자동차
3. 전원 공급 장치의 출력을 5V로 유지하는 전압 조정기(voltage regulator)

### 연습 2: 개루프 vs. 폐루프

1. 세탁기가 고정된 30분 사이클로 돌아간다. 이것은 개루프인가, 폐루프인가? 어떻게 하면 폐루프로 만들 수 있는가?
2. 경사진 도로에서 자동차의 크루즈 컨트롤이 순수한 개루프 방식으로는 작동할 수 없는 이유를 설명하라.
3. 폐루프 제어보다 개루프 제어가 선호되는 예를 하나 들어라.

### 연습 3: 피드백의 효과

플랜트 이득 $G_p$가 8에서 12 사이에서 변할 수 있는 시스템을 고려하자 (공칭값(nominal) = 10).

1. 개루프에서, 전체 이득이 1이 되도록 제어기 이득을 $G_c = 0.1$로 설정한다. 출력 이득의 범위는 얼마인가?
2. 폐루프(단위 피드백 $H = 1$)에서 $G_c = 100$일 때, $G_p = 8, 10, 12$에 대해 폐루프 이득 $\frac{G_c G_p}{1 + G_c G_p}$를 계산하라. 출력이 얼마나 변하는가?
3. 민감도를 줄이기 위해 높은 루프 이득을 사용할 때의 대가(cost)는 무엇인가?
